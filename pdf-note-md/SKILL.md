# 目标
围绕“转为MD(copilot读不了pdf)”这一类任务，稳定执行原 prompt 想表达的核心能力，并把零散命令沉淀为可复用的 skill。

## 适用场景
- 当需要把当前笔记或 PDF 转录内容完整转换为 Markdown，同时保留原有层级结构且不遗漏信息时使用。
- 适合需要复用同一类提示词、同时希望 agent 按固定结构稳定输出结果的场景。

## 输入要求
- `activeNote` 或当前笔记内容

## 输出要求
- 输出需为 Markdown 结构，保持标题、列表、代码块等格式要求

## 执行步骤
1. 先识别输入材料、上下文来源和隐含占位符，明确本次处理对象。
2. 提取原 prompt 的核心任务、结构约束和禁止项，再将它们转写为稳定的执行顺序。
3. 在最终输出前逐项校验格式、语气、长度和保留项，避免漏掉原 prompt 的关键约束。

## 保留约束
以下原始约束在执行时必须继续遵守：

```markdown
将{activeNote}完整转换为 Markdown 格式，保持原有的层级结构，不要遗漏任何信息，且不要添加额外的解释。
```

## 注意事项
- 将 `{}`、`{activeNote}` 一类占位符视为运行时输入，不要把占位符原样遗留在最终结果里。
- 原 prompt 禁止额外解释时，不能附带说明、免责声明或过程暴露。
- 凡是要求保留语气、结构、换行或特殊字符的场景，优先保真，避免不必要重写。
