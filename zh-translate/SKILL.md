# 目标
围绕“翻译为中文”这一类任务，稳定执行原 prompt 想表达的核心能力，并把零散命令沉淀为可复用的 skill。

## 适用场景
- 当需要把文本翻译为中文，并同时保留原始意义、语气、文化语境以及格式结构时使用。
- 适合需要复用同一类提示词、同时希望 agent 按固定结构稳定输出结果的场景。

## 输入要求
- 用户显式提供的正文、片段或待处理文本

## 输出要求
- 最终回复只输出结果本体，不额外附加解释、前言或注释
- 译文需保留原文意义、语气和结构

## 执行步骤
1. 先识别输入材料、上下文来源和隐含占位符，明确本次处理对象。
2. 提取原 prompt 的核心任务、结构约束和禁止项，再将它们转写为稳定的执行顺序。
3. 翻译时先保证语义准确，再兼顾语气、文化语境和版式一致性。
4. 输出前做一次格式自检，确保最终回复中没有多余解释。
5. 在最终输出前逐项校验格式、语气、长度和保留项，避免漏掉原 prompt 的关键约束。

## 保留约束
以下原始约束在执行时必须继续遵守：

```markdown
将{}翻译成中文：
1.保留意义和语气
2.保持适当的文化背景
3.保持格式和结构
仅返回翻译后的文本。
```

## 注意事项
- 将 `{}`、`{activeNote}` 一类占位符视为运行时输入，不要把占位符原样遗留在最终结果里。
- 凡是要求保留语气、结构、换行或特殊字符的场景，优先保真，避免不必要重写。
