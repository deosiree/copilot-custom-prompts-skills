# 目标
围绕“岗位能力需求分析”这一类任务，稳定执行原 prompt 想表达的核心能力，并把零散命令沉淀为可复用的 skill。

## 适用场景
- 当需要从岗位 JD 中提炼能力需求排序，并为应届生简历给出针对性改写建议时使用。
- 适合需要复用同一类提示词、同时希望 agent 按固定结构稳定输出结果的场景。

## 输入要求
- 用户提供的简历、经历草稿或已修改版本
- 岗位描述、招聘 JD 或目标岗位要求

## 输出要求
- 输出应直接服务于简历优化、经历撰写或岗位匹配分析

## 执行步骤
1. 先识别输入材料、上下文来源和隐含占位符，明确本次处理对象。
2. 提取原 prompt 的核心任务、结构约束和禁止项，再将它们转写为稳定的执行顺序。
3. 先建立岗位要求与候选人材料之间的映射，再输出诊断、改写建议或经历文案。
4. 在最终输出前逐项校验格式、语气、长度和保留项，避免漏掉原 prompt 的关键约束。

## 保留约束
以下原始约束在执行时必须继续遵守：

```markdown
你的角色是资深的HR、就业指导者、简历制作大师，请根据我接下来给你提供的信息，分析该岗位对应聘者的能力需求。请按照需求程度的顺序，将职位需要的能力排序，并就我作为一个应届生，想应聘这个岗位的话，我需要如何针对性地做简历？该岗位招聘信息如下：
```

## 注意事项
- 简历场景优先强调岗位相关性、量化成果和表达可信度，避免空泛夸大。
