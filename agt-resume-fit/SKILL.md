# 目标
围绕“简历优化（agent）”这一类任务，稳定执行原 prompt 想表达的核心能力，并把零散命令沉淀为可复用的 skill。

## 适用场景
- 当需要基于当前笔记评估向 Agent 开发方向转型的简历缺口，并通过追问挖掘潜在项目经验时使用。
- 适合需要复用同一类提示词、同时希望 agent 按固定结构稳定输出结果的场景。

## 输入要求
- `activeNote` 或当前笔记内容

## 输出要求
- 输出应完整满足原 prompt 中声明的格式与结构约束

## 执行步骤
1. 先识别输入材料、上下文来源和隐含占位符，明确本次处理对象。
2. 提取原 prompt 的核心任务、结构约束和禁止项，再将它们转写为稳定的执行顺序。
3. 如果任务包含追问或对话推进，按原 prompt 设定的节奏逐步互动，而不是一次性给完。
4. 在最终输出前逐项校验格式、语气、长度和保留项，避免漏掉原 prompt 的关键约束。

## 保留约束
以下原始约束在执行时必须继续遵守：

```markdown
先根据{activeNote}，给出初步的转型分析，并列出你认为我必须补充到简历中的“Agent 开发硬技能”。随后开始向我提问，挖掘我的潜在项目经验。
```

## 注意事项
- 将 `{}`、`{activeNote}` 一类占位符视为运行时输入，不要把占位符原样遗留在最终结果里。
