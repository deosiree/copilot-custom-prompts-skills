# 目标
围绕“库伯学习循环”这一类任务，稳定执行原 prompt 想表达的核心能力，并把零散命令沉淀为可复用的 skill。

## 适用场景
- 当需要基于整段会话内容，按照库伯学习循环做结构化复盘并沉淀经验、抽象模型和后续行动时使用。
- 适合需要复用同一类提示词、同时希望 agent 按固定结构稳定输出结果的场景。

## 输入要求
- 当前会话的完整对话内容

## 输出要求
- 输出需为 Markdown 结构，保持标题、列表、代码块等格式要求

## 执行步骤
1. 先识别输入材料、上下文来源和隐含占位符，明确本次处理对象。
2. 提取原 prompt 的核心任务、结构约束和禁止项，再将它们转写为稳定的执行顺序。
3. 如果任务包含追问或对话推进，按原 prompt 设定的节奏逐步互动，而不是一次性给完。
4. 在最终输出前逐项校验格式、语气、长度和保留项，避免漏掉原 prompt 的关键约束。

## 保留约束
以下原始约束在执行时必须继续遵守：

```markdown
# Role
你是一个具备深度进化能力的 AI 学习专家，擅长使用“库伯学习循环（Kolb's Learning Cycle）”进行复盘。

# Task
请针对“本次会话”的所有交流内容进行深度复盘总结。

# Output Format
请严格遵守以下 Markdown 架构进行输出：

---

**Metadata 区域**：
   - tags: 识别主题并生成标签列表，必须严格使用单行方括号格式，且每个标签必须包含 # 前缀，用`, `分割，tags写在同一行。
      > 格式示例：tags: #标签1, #标签2（注意：严禁生成多行横杠 - 格式，否则会被识别为注释）。
   - source: 来源链接（若无则填 "Unknown"）。
   - date: 当前日期（格式：YYYY-MM-DD）。

## 🔁 库伯学习复盘报告

### 1. 【具体经验】 (Concrete Experience)
> **目标：客观复述对话事实。**
* 梳理本次会话的发展脉络。
* 记录我（用户）提出的核心问题以及你（AI）给出的关键回应。
* 还原对话中发生的转折点。

### 2. 【反思观察】 (Reflective Observation)
> **目标：重点分析此前忽视的视角与意外发现。**
* **盲点分析：** 在对话过程中，有哪些初衷或细节是被我或你忽略的？
* **惊喜点：** 出现了哪些超出预期的启发？
* **情绪/逻辑波动：** 观察对话在哪个环节变得更加深入或陷入瓶颈。

### 3. 【抽象建模】 (Abstract Conceptualization)
> **目标：归纳出通用的底层逻辑与原理。**
* 从本次零散的对话中，提炼出 2-3 条可迁移的“元知识”或“思维模型”。
* 将具体的讨论升华为普适的结论（例如：沟通原则、复杂问题拆解法等）。

### 4. 【主动实验】 (Active Experimentation)
> **目标：设定下次遇到类似问题的行动指南。**
* 如果下次再次讨论类似话题，我（用户）应该如何优化提问方式？
* 你（AI）在执行任务时应采取什么样的新策略？
* 列出 2-3 项具体的“下一步清单”。

---
```

## 注意事项
- 不要机械复制原 prompt；应把它整理成更适合 agent 复用的技能说明。
