# 目标
围绕“简历-项目经历撰写”这一类任务，稳定执行原 prompt 想表达的核心能力，并把零散命令沉淀为可复用的 skill。

## 适用场景
- 当需要根据已有项目信息撰写更有行动力、量化结果更充分的项目经历时使用。
- 适合需要复用同一类提示词、同时希望 agent 按固定结构稳定输出结果的场景。

## 输入要求
- 与该 prompt 对应的原始输入材料

## 输出要求
- 输出应完整满足原 prompt 中声明的格式与结构约束

## 执行步骤
1. 先识别输入材料、上下文来源和隐含占位符，明确本次处理对象。
2. 提取原 prompt 的核心任务、结构约束和禁止项，再将它们转写为稳定的执行顺序。
3. 在最终输出前逐项校验格式、语气、长度和保留项，避免漏掉原 prompt 的关键约束。

## 保留约束
以下原始约束在执行时必须继续遵守：

```markdown
基于我为你提供的信息，按照以下要求，撰写项目经历：
（1）逻辑清晰，通顺易读。将长难句拆分成短句来增加可读性。
（2）多使用动词开头：使用强有力的动词（如“领导”、“开发”、“实施”等）来开头，强调你的主动性和影响力。
（3）尽可能地量化成果：尽量用数据和具体成果来展示你的贡献，例如“提高团队效率20%”或“成功协调5个跨部门项目”。
（4）技能强调：突出在项目中运用的技能，能为公司解决的问题，带来什么结果。
```

## 注意事项
- 不要机械复制原 prompt；应把它整理成更适合 agent 复用的技能说明。
