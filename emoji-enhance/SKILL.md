# 目标
围绕“emoji化”这一类任务，稳定执行原 prompt 想表达的核心能力，并把零散命令沉淀为可复用的 skill。

## 适用场景
- 当需要在不改动原文内容的前提下，按自然断点加入符合语境的表情符号时使用。
- 适合需要复用同一类提示词、同时希望 agent 按固定结构稳定输出结果的场景。

## 输入要求
- 用户显式提供的正文、片段或待处理文本

## 输出要求
- 最终回复只输出结果本体，不额外附加解释、前言或注释

## 执行步骤
1. 先识别输入材料、上下文来源和隐含占位符，明确本次处理对象。
2. 提取原 prompt 的核心任务、结构约束和禁止项，再将它们转写为稳定的执行顺序。
3. 输出前做一次格式自检，确保最终回复中没有多余解释。
4. 在最终输出前逐项校验格式、语气、长度和保留项，避免漏掉原 prompt 的关键约束。

## 保留约束
以下原始约束在执行时必须继续遵守：

```markdown
添加相关表情符号以增强{}。遵循以下规则：
1.在文本中的自然中断处插入表情符号
2.切勿将两个表情符号放在一起
3.保持所有原文不变
4.选择符合上下文和语气的表情符号
仅返回表情符号文本。
```

## 注意事项
- 将 `{}`、`{activeNote}` 一类占位符视为运行时输入，不要把占位符原样遗留在最终结果里。
