# 目标
围绕“转为ER图”这一类任务，稳定执行原 prompt 想表达的核心能力，并把零散命令沉淀为可复用的 skill。

## 适用场景
- 当需要从内容中抽取实体及其关系，并输出 Mermaid ER 图代码块时使用。
- 适合需要复用同一类提示词、同时希望 agent 按固定结构稳定输出结果的场景。

## 输入要求
- 用户显式提供的正文、片段或待处理文本

## 输出要求
- 若要求生成 Mermaid，必须只输出合法的 Mermaid 代码块

## 执行步骤
1. 先识别输入材料、上下文来源和隐含占位符，明确本次处理对象。
2. 提取原 prompt 的核心任务、结构约束和禁止项，再将它们转写为稳定的执行顺序。
3. 在生成图表前先判断最合适的 Mermaid 图类型，再按语法限制产出代码块。
4. 在最终输出前逐项校验格式、语气、长度和保留项，避免漏掉原 prompt 的关键约束。

## 保留约束
以下原始约束在执行时必须继续遵守：

```markdown
分析{}，并将其逻辑结构转化为 Mermaid.js 代码。
要求：
1.使用 ER 图 (Entity Relationship Diagram) 表现实体间的对应关系。
2.节点名称要简洁，最好使用中文，并使用子图 (subgraph) 对相关模块进行分组。
请只输出 Mermaid 代码块，不要包含任何多余的解释。
```

## 注意事项
- 将 `{}`、`{activeNote}` 一类占位符视为运行时输入，不要把占位符原样遗留在最终结果里。
- Mermaid 场景优先保证语法正确，其次再考虑节点命名美观。
