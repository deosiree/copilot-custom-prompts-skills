# 目标
围绕“撰写工作经历”这一类任务，稳定执行原 prompt 想表达的核心能力，并把零散命令沉淀为可复用的 skill。

## 适用场景
- 当需要根据岗位 JD 提炼招聘需求关键词，并用 TMR 结构定向撰写工作经历与整份简历时使用。
- 适合需要复用同一类提示词、同时希望 agent 按固定结构稳定输出结果的场景。

## 输入要求
- 用户提供的简历、经历草稿或已修改版本
- 岗位描述、招聘 JD 或目标岗位要求

## 输出要求
- 输出应直接服务于简历优化、经历撰写或岗位匹配分析

## 执行步骤
1. 先识别输入材料、上下文来源和隐含占位符，明确本次处理对象。
2. 提取原 prompt 的核心任务、结构约束和禁止项，再将它们转写为稳定的执行顺序。
3. 若原 prompt 已给出分步流程，按其步骤顺序执行，不要打乱阶段目标。
4. 先建立岗位要求与候选人材料之间的映射，再输出诊断、改写建议或经历文案。
5. 在最终输出前逐项校验格式、语气、长度和保留项，避免漏掉原 prompt 的关键约束。

## 保留约束
以下原始约束在执行时必须继续遵守：

```markdown
我需要你扮演一名职位描述与简历匹配专家。我会将招聘的JD发送给你，你的任务分为三步：

STEP1：基于JD，你需要提炼出每个预投岗位的招聘需求，例如，通过日常运营和活动策划，有效提升社群用户的活跃和转化率。该岗位JD的招聘需求关键词是：提升社群用户的【活跃】和【转化率】。你需要提供的是招聘需求关键词。

STEP2：基于每个岗位的招聘需求关键词，你需要使用TMR（任务-方法-结果）描述法撰写工作经历。例如，TMR描述法的使用：T（Task）任务，如，负责用户增长。M（Method）方法，如，通过实验内容营销、搜索营销和社群营销。R（Result）业绩结果，如，实现了单月25万的用户新增。你的回答必须以这种方式形成一条完整的工作经历。

STEP3：针对每个招聘需求关键词，你都要如此造句。由此，拼凑成一份有针对性的简历。你的回答应以这种方式构建完整的简历。我的第一份JD是'负责社区运营和活动策划，以提高用户活跃度和转化率'。请给出你的答复。
```

## 注意事项
- 简历场景优先强调岗位相关性、量化成果和表达可信度，避免空泛夸大。
