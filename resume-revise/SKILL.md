# 目标
围绕“修改简历”这一类任务，稳定执行原 prompt 想表达的核心能力，并把零散命令沉淀为可复用的 skill。

## 适用场景
- 当需要围绕简历诊断、岗位匹配分析和格式化润色，分阶段改写简历内容时使用。
- 适合需要复用同一类提示词、同时希望 agent 按固定结构稳定输出结果的场景。

## 输入要求
- 用户提供的简历、经历草稿或已修改版本
- 岗位描述、招聘 JD 或目标岗位要求

## 输出要求
- 若原 prompt 要求表格，输出需整理为清晰的表格结构
- 输出应直接服务于简历优化、经历撰写或岗位匹配分析

## 执行步骤
1. 先识别输入材料、上下文来源和隐含占位符，明确本次处理对象。
2. 提取原 prompt 的核心任务、结构约束和禁止项，再将它们转写为稳定的执行顺序。
3. 先建立岗位要求与候选人材料之间的映射，再输出诊断、改写建议或经历文案。
4. 在最终输出前逐项校验格式、语气、长度和保留项，避免漏掉原 prompt 的关键约束。

## 保留约束
以下原始约束在执行时必须继续遵守：

```markdown
1.上传简历，提示词：请你作为资深HR，找出我简历存在的主要问题，怎么说我都能接受，可以用表格形式呈现。
2.复制应聘岗位要求，提示词：这是我想面试的公司的岗位要求，请你用HR视觉分析我的简历里的工作经验部分，如何将我的优势、实力展现出来？用表格形式。
3.上传修改后简历，提示词：请帮我把这份简历梳理成专业简历的格式。
```

## 注意事项
- 简历场景优先强调岗位相关性、量化成果和表达可信度，避免空泛夸大。
